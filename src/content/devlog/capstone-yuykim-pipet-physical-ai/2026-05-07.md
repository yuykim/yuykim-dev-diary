# 오늘 한 일

- `state_profile=extended` 26D 입력의 빈 슬롯 문제 발견 및 수정
- Pinocchio FK로 TCP 위치(ee_pose) 계산 후 state에 채워 넣기
- `26.05.03_half_data_100` (100ep, 35,422 frame) 변환 완료
- CUDA OOM 발생 → batch_size=8로 재실행 명령 확정

# 왜 이 작업을 했는가

`--state_profile extended` 26D 중 뒤 8칸(ee_pose + gripper_state)이 실제로는 0으로 패딩되어 있었음. 모델이 가짜 입력으로 학습된 것 → 수정 필요.

# 작업 과정

**변환 단계 보강(`convert.py`):**
- `gripper_state` 없으면 `gripper_actions[t]` 이산값으로 대체
- `ee_pose` 없으면 Pinocchio FK(URDF + joint positions)로 TCP 위치 계산 → 7D 채움
- 신규 모듈: `indy7_tcp_fk.py`

**추론 단계 보강(`inference_node.py`):**
- `state_target_dim >= 26`이면 TF lookup으로 ee_pose 채우고, 실패 시 Pinocchio FK로 fallback
- `observation.state[25]`는 마지막 그리퍼 명령 기록

**100ep 데이터셋:**
- 입력: `26.05.03_half_data_100` 100ep, 28GB
- 출력: `ai/datasets/pipet_extended_depth_100` (15GB, 35,322 frames, fps=20)
- features: `front`, `front_depth`, `overhead`, `overhead_depth`, `state(26D)`, `action(7D)`

**CUDA OOM:** batch=16 → OOM (RTX 3080 Laptop 8GB). batch=8로 재실행 → 정상.
