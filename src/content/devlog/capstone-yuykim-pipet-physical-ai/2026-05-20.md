# 오늘 한 일

- DAgger AUTO pose republish 보강 (토픽 갱신 안 되는 문제 해결)
- Operator GUI 추가 (`operator_gui.py`)
- 운용용 shell alias 정리 (`~/.bashrc`)
- `act_remove_60_cartesian_360` 090000 실기 실행 alias 추가

# 왜 이 작업을 했는가

AUTO 상태에서 `/indy/teleop_pose`가 갱신 안 되는 현상 → 모델 output이 있어도 로봇이 안 움직임. GUI와 alias가 없으면 매번 긴 명령을 직접 입력해야 해서 실험 속도가 느려짐.

# 작업 과정

**AUTO pose republish:**
- 모델 callback에만 의존하면 타이밍 문제로 갱신 안 될 수 있음
- AUTO 상태에서 마지막 모델 pose를 control loop에서 계속 republish하도록 보강
- TAKEOVER 중에는 인간 조작 pose만 publish

**Operator GUI:**
- `/wrist_camera`, `/overhead_camera` 실시간 표시
- `A` 또는 Start 버튼: ZMQ 서버 + inference_ros 동시 시작
- `H` 또는 Home 버튼: `move_home.py --stop-teleop` 실행

**shell alias (`~/.bashrc`):**
- `camview`: ROS2 + rqt_image_view
- `home`: `move_home.py --stop-teleop --stop-before --servo-on`
- `pad_control`: xbox_servo_node
- `backend`: data_collection.launch.py
- `robo_backend`: ZMQ ACT 서버 시작
- `ai_control`: cartesian inference 실행

**090000 실기 설정:**
- `control_mode=cartesian`, `state_target_dim=18`, `image_target_height=360`
- `max_cartesian_speed_mm_s=40.0`, `max_delta_mm=1.0`

# 중간 결과

- DAgger AUTO forwarding 안정화
- GUI + alias로 실험 진행이 훨씬 빨라짐
