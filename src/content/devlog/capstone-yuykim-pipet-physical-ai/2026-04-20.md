# 오늘 한 일

- 10개 NPZ 에피소드 전수 검증 및 LeRobot v3 변환
- 100k 스텝 학습 실행 (`ai/lerobot/train_26_04_20_half_data.sh`)
- 하드웨어 점검 후 추론 시도
- 파이펫 위치 인식 실패 원인 분석 및 보완 방향 검토

# 왜 이 작업을 했는가

적은 데이터로 전체 파이프라인(수집→변환→학습→추론)이 제대로 돌아가는지 검증하기 위해.

# 작업 과정

**데이터 검증:** 10개 NPZ 전수 확인 (필수 키, 배열 길이, success=True, 손상 없음). 에피소드 길이 15.8~20.6s, 약 17.6~20Hz.

**변환:** `repo_id=pipet_26_04_20_half_data`, 스모크 테스트: episodes=10, frames=3448, fps=20 정상.

**하드웨어 점검:**
- Mark7 포트 `/dev/ttyACM0` → `/dev/ttyACM1`로 변경되어 있음 → `mark7_port:=/dev/ttyACM1`로 해결
- RealSense 2대: USB 재열거 오류 → 재기동 후 정상 (~30Hz)

**파이펫 위치 인식 실패 분석:**
- 마지막 2~5cm 정밀 접근 불안정
- grasp 명령이 너무 빨리 나옴
- LLM/VLM 보조 판단기 아이디어: 저속 주기(0.5~1Hz)로 파이펫 방향/stage 판단

# 중간 결과

- 파이프라인 전 구간 정상 동작 확인
- 학습 완료: `ai/models/26.04.20_half_data/checkpoints/last`
- 추론 기본 실행 메모: Mark7 포트는 항상 실제 장치 노드로 지정

# 막힌 점

- Mark7 포트 번호가 매번 달라질 수 있어 실행 전 확인이 번거로움
- 카메라 `Publisher count: 0`이면 스트림 실패 상태 → 재기동 필요

# 느낀 점

10개 에피소드만으로는 위치 일반화가 어렵다. 데이터 양보다 수집 퀄리티(파이펫 위치 다양성, 조명, 접근 경로 다양성)가 더 중요할 수 있다.
