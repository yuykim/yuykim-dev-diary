# 오늘 한 일

- grasp-focus 학습 20k 체크포인트 상태 확인
- `run_lerobot_train.py`에 `--resume` 인자 지원 추가
- `num_workers=4`로 dataloader 병목 개선

# 왜 이 작업을 했는가

학습 도중 `--num_workers 2`로 돌리다가 `data_s`가 `updt_s`의 10배라는 걸 발견. dataloader 병목이 학습 시간을 크게 늘리고 있었다. 또한 기존 wrapper가 `--resume`을 지원하지 않아 20k에서 이어서 돌릴 수 없었음.

# 작업 과정

**dataloader 병목 확인:**
- `updt_s ≈ 0.24s`, `data_s ≈ 2.4~2.6s` → GPU 계산보다 데이터 로딩이 10배 느림
- 원본 100ep 학습은 `num_workers=4`로 안정적이었으므로 동일 설정으로 이어가기로

**resume 지원 추가:**
- `run_lerobot_train.py`에 `--resume`, `--resume_config_path` 인자 추가
- LeRobot 내부 parser는 `--config_path=경로` 형태만 인식 → `=` 연결 형태로 전달
- 기본 resume config 경로: `<output_dir>/checkpoints/last/pretrained_model/train_config.json`

# 중간 결과

- 20k 체크포인트에서 resume 성공, step이 20k 근처에서 이어짐
- `num_workers=4` 적용 후 `data_s` 개선

# 막힌 점

- LeRobot `parse_arg("config_path")`가 `--config_path 경로` (공백) 형태를 못 읽음 → `--config_path=경로` 형태로 해결

# 느낀 점

wrapper를 만들 때 resume을 처음부터 고려했어야 했다. 학습이 길어질수록 resume 기능은 필수다.
