# 오늘 한 일

- `inference_node`에 `control_mode=cartesian` 추가
- `move_home.py` 단독 스크립트 추가
- Cartesian 추론 state 입력 정합 수정 (ee_pose 18D)
- remove grasp-focus 증강 데이터 생성 (60+60=120 ep)

# 왜 이 작업을 했는가

`action_space=cartesian`으로 변환/학습한 모델은 `action[:6]`이 관절 delta가 아니라 `delta_ee_pose`다. 기존 `inference_node`가 관절 delta로 해석하면 의미가 완전히 달라진다.

# 작업 과정

**Cartesian 추론 모드:**
- `control_mode=joint`: 기존 유지 (`action[:6] → delta_q`)
- `control_mode=cartesian`: `action[:6] → delta_ee_pose`, `/indy/teleop_pose`로 publish, `indy_srv data=5`
- 안전 클립: `max_delta_mm`, `max_delta_deg`, `max_cartesian_speed_mm_s`, `max_angular_speed_deg_s`

**state 입력 정합:**
- `state_target_dim=18`일 때 기존 `[q, dq, tau]`가 아니라 `[q, dq, ee_pose]`로 구성
- `/indy/ee_pose` 토픽 구독으로 실제 TCP 위치 채움

**move_home.py:**
- ROS 없이 `IndyDCP3.movej()`로 직접 home 이동
- home: `[0, 25, -115, 90, 0, 0] deg`
- `--stop-before`, `--recover`, `--servo-on` 옵션

**grasp-focus 증강:**
- 기존 remove 성공 60개에서 grasp 기준 3s/2s window crop → 60 focus ep
- 결과: 120 ep (원본 60 + crop 60)

# 중간 결과

- Cartesian 추론 모드 코드 완성
- state 입력 분포 일치 확인
- 120 ep 데이터셋 생성 완료

# 막힌 점

- 기존 `state_target_dim=18`이 `[q, dq, tau]`를 기준으로 짜여 있어 수정 범위가 넓었음
- move_home 직후 teleop 모드와 충돌 방지를 위해 `--stop-teleop` 옵션 추가 필요

# 느낀 점

Cartesian 제어가 관절 제어보다 사람의 직관에 훨씬 가깝다. 학습 데이터도 Cartesian으로 수집하면 분포가 더 자연스러울 것 같다.
