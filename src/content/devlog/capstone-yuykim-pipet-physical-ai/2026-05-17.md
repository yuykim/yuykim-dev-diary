# 오늘 한 일

- Xbox 컨트롤러 기반 Indy7+Mark7 텔레옵 노드 추가
- Neuromeka IndyDCP3 Python API 정리 문서 작성
- 데이터 수집 스키마/학습 방향 최종 결정
- `joint_efforts` 수집 제외 결정
- Xbox 2단계 수집 흐름 추가 (remove → insert)

# 왜 이 작업을 했는가

키보드보다 Xbox 패드가 Cartesian 이동에 훨씬 자연스럽다. 또한 데이터 스키마를 명확히 정하지 않으면 변환/학습마다 혼란이 생긴다.

# 작업 과정

**Xbox 텔레옵:**
- 새 노드: `xbox_servo_node.py`
- pygame joystick 백엔드가 불안정 → `linuxevdev` 기본으로 설정
- D-pad(X/Y), LT/RT(Z), right stick(Rx/Ry), LB/RB(Rz)
- A/B/X/Y: grasp/open/press/release
- START: 녹화 시작/종료, A/B/X: success/fail/discard

**데이터 스키마 확정:**
- 필수 저장: `timestamps`, `joint_positions`, `ee_poses`, `wrist/overhead_rgb_images`, `gripper_actions`
- 학습 action: `delta_ee_pose(6) + gripper_action(1)` (Cartesian 기반)
- `joint_efforts`: `/joint_states.effort`가 `[]`로 publish됨 → 수집 제외

**2단계 수집:** remove 성공 → insert 수집 여부 질의 → `episodes/remove/`, `episodes/insert/`로 분리 저장
