# 오늘 한 일

- LeRobot ACT 체크포인트로 실기 추론 시도
- Indy7 팔이 움직이지 않는 원인 분석
- 학습 데이터의 `delta_q` 분포 통계 확인

# 왜 이 작업을 했는가

수집한 데이터로 학습한 첫 모델을 실제 로봇에 올려보기 위해.

# 작업 과정

- ZMQ 사이드카(`zmq_act_server`) + `inference.launch.py`로 추론 실행
- `ros2 topic info /joint_trajectory_controller/joint_trajectory -v` 로 토픽 연결 확인 → Publisher(inference_node) 1개, Subscription(indy_driver) 1개 → 연결 자체는 정상
- NPZ 파일에서 `delta_q` 통계 추출:
  - 전체 스텝 중 약 52%가 "전 관절 |dq| < 1e-4" (정지에 가까움)
  - 6번째 축(인덱스 5)은 거의 안 움직임 (max ~1e-5 rad)

# 중간 결과

- 팔이 거의 안 움직이는 건 "모델이 학습 못함"이 아니라 **데이터 자체가 스텝당 델타가 작은 분포**였기 때문
- LeRobot의 `predict_action`은 추론 시 역정규화(비정규화)를 제대로 하고 있음 → 정규화 문제 아님
- Mark7 그리퍼는 반응함 → 팔만 약하게 움직이는 패턴

# 막힌 점

- `delta_norm`이 ~0.001 수준으로 너무 작아 실제 체감 움직임이 없음
- 직접 교시(Direct teaching) ON 상태에서 추론하면 드라이버가 궤적 명령을 무시할 수 있음

# 느낀 점

데이터 수집 품질이 결국 모든 것을 결정한다. 정지 구간이 너무 많으면 모델이 "가만히 있는 것"을 학습한다. 다음 수집은 정지 구간을 최소화해야 함.
