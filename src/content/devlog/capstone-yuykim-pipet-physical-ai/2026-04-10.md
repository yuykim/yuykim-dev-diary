# 오늘 한 일

- `autonomy_enabled:=true`에서 Indy7 팔이 전혀 안 움직이는 문제 해결
- 추론 이미지 해상도·delta scale 파라미터 추가
- ZMQ 사이드카 실행 순서 정리
- `ai/` 디렉토리 구조 정리 (`datasets/`, `models/`, `logs/`)

# 왜 이 작업을 했는가

`050000` 체크포인트로 실기 추론 시도했더니 팔이 전혀 반응하지 않음. 원인을 찾아 수정해야 실제 성능 평가가 가능.

# 작업 과정

**팔 무반응 원인:** Indy driver의 `joint_trajectory_callback`은 내부 상태가 `MSG_TELE_JOINT_ABS`일 때만 궤적을 처리함. 데이터 수집 후 direct teaching 상태가 남아 있으면 드라이버가 궤적 명령 무시. → `inference_node`가 추론 시작 전 `indy_srv data=6`을 자동 호출하도록 수정.

**delta_norm 작음:** `act_360_idle` 모델은 `360×480` 이미지로 학습했는데 추론은 `640×480` RealSense 그대로 넣으면 분포 불일치 → `image_target_height/width` 파라미터 추가. `action_delta_scale` 파라미터도 추가.

**디렉토리 정리:**
- `ai/datasets/`: LeRobotDataset
- `ai/models/`: 체크포인트
- `ai/logs/`: 학습 로그

# 중간 결과

- `Indy 관절 텔레옵 준비 완료 (indy_srv data=6)` 로그 확인 후 팔 명령 정상 전달
- `action_delta_scale:=8.0`, `image_target_height:=360`, `image_target_width:=480` 추가 설정으로 체감 움직임 개선
- Mark7 그리퍼 플리킹(open↔grasp 반복) 관찰 → 히스테리시스 추가 권장 메모

# 막힌 점

- `050000` 체크포인트가 `ai/huggingFace/lerobot` 경로를 물고 있어 `ModuleNotFoundError: lerobot.configs.policies` → `ai/lerobot_source/lerobot`으로 editable 재설치
- Mark7 포트가 `/dev/ttyACM0` → `/dev/ttyACM1`로 바뀌는 경우가 있음 → 실행 전 확인 필요

# 느낀 점

"왜 안 되지?"를 계속 파다 보면 결국 소프트웨어 아키텍처가 머릿속에 명확히 그려진다. 드라이버 내부 상태 머신을 이해하고 나니 비슷한 문제를 훨씬 빠르게 찾을 수 있을 것 같다.
